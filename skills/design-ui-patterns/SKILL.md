# Skill: design-ui-patterns

## 目的

UI/UX 設計時に参照する、ライブラリ非依存の設計原則とパターン集。
プロジェクト固有のコンポーネントライブラリは `.claude/config/ui.yml` で別途指定する。

## 設計原則

### 1. 一貫性

- 既存画面と同じインタラクションパターンを使う
- ボタン・フォーム・モーダルなどの要素は既存コンポーネントを再利用
- カラー・スペーシング・タイポグラフィはデザイントークンから

### 2. フィードバック

ユーザー操作には必ず視覚的フィードバックを返す:

| 操作 | フィードバック |
|------|--------------|
| ボタン押下 | hover / pressed 状態、リップル等 |
| 非同期処理 | ローディングインジケータ、ボタン無効化 |
| 成功 | トースト・確認画面・該当箇所のハイライト |
| エラー | エラーメッセージ（具体的な原因と次の行動） |
| 入力中 | リアルタイムバリデーション（ブラー時または入力中） |

### 3. 状態の表現

各画面・コンポーネントが取り得る状態を全て設計する:

- **Empty**: データが0件のとき
- **Loading**: 取得中
- **Loaded**: 通常表示
- **Error**: 取得失敗
- **Partial**: 一部失敗・ページネーション中

### 4. アクセシビリティ

- すべてのインタラクティブ要素にキーボード操作を提供
- フォーカス順序が論理的
- 画像・アイコンに代替テキスト
- カラーコントラスト比 WCAG AA 以上

### 5. レスポンシブ

ブレークポイントを意識した3層設計:

- **モバイル**（< 768px）: 縦積み、ボトムナビゲーション
- **タブレット**（768〜1024px）: 2カラムまで
- **デスクトップ**（> 1024px）: 多カラム、サイドバー

## 典型パターン

### フォーム

```
- ラベル: フィールドの上に配置（左配置はモバイルで幅を取る）
- 必須マーク: ラベル横に *
- ヘルプテキスト: フィールドの下に小さく
- バリデーション: ブラー時に検証、エラーはフィールド下に赤字
- 送信ボタン: フォーム末尾、無効状態を視覚的に区別
- 成功時: トーストまたは画面遷移
```

### リスト

```
- 検索 / フィルタ: リスト上部
- 並び替え: カラムヘッダクリック or 専用 UI
- ページネーション: リスト下部 or 無限スクロール
- 選択: チェックボックス、選択中の件数を表示
- アクション: 選択 → 一括操作ボタン表示
```

### 確認ダイアログ

破壊的操作（削除・公開取り消し等）には必ず確認を挟む:

```
- タイトル: 操作の名前
- 本文: 操作の影響を具体的に説明（「N件のデータが削除されます」等）
- ボタン: 「キャンセル」を左、「実行」を右、破壊的操作は赤
- 危険度が高い場合はチェックボックス確認や入力確認を追加
```

### 通知

```
- トースト: 成功・軽微なエラーに使用、自動消滅
- バナー: 重要な情報、ユーザーが閉じるまで表示
- モーダル: 操作の確認、エラー詳細
- インライン: フォームのバリデーション等、該当箇所の近く
```

## 出力時のチェックリスト

- [ ] 全ての状態（Empty / Loading / Error 含む）が定義されている
- [ ] 各操作に視覚的フィードバックがある
- [ ] レスポンシブ挙動が明示されている
- [ ] アクセシビリティ配慮（キーボード・代替テキスト等）が記述されている
- [ ] 既存コンポーネントの再利用箇所が示されている
