# ローカルタスク管理リファレンス

`mode.tasks: local` の場合に Conductor / todo が参照する。

## タスクファイル

場所: `docs/tasks.md`（プロジェクトルート相対）

### フォーマット

```markdown
# Tasks

次のタスク番号: 4

## Triage
- #1 タスクタイトル
  説明やメモ

## Todo
- #2 タスクタイトル
  説明やメモ

## In Progress
- #3 タスクタイトル
  説明やメモ

## In Review

## Done
- #4 タスクタイトル
  完了時のメモ
```

### ルール

- タスク番号は `#` + 連番。「次のタスク番号」を参照してインクリメントする
- 各セクションが GitHub Project のステータスに対応する
- ステータス変更はタスク行を別のセクションに移動することで行う
- 説明は任意。タスク行の次行にインデント付きで記述する

## タスク操作

### 追加

1. 「次のタスク番号」の値で `#N タイトル` を該当セクション（通常 `Triage` または `Todo`）に追記
2. 「次のタスク番号」をインクリメント

### 参照

`/run #N` で指定された番号を `docs/tasks.md` から検索し、タイトルと説明を読み取る。

### ステータス更新

タスク行を移動先セクションに切り取り＆貼り付けする。

例: `#3` を `In Progress` → `Done` に移動する場合、`## In Progress` セクションから行を削除し、`## Done` セクションに追記する。

## ステータスフロー

GitHub モードと同じフローを使用する:

```
Triage → Todo → In Progress → In Review → Done
```

| ステータス | 意味 | 遷移タイミング |
|-----------|------|--------------|
| Triage | 要望・検討中 | 要望を受けた時点 |
| Todo | 具体化済み・着手待ち | 要件が明確になった時点 |
| In Progress | 作業中 | 実装開始時 |
| In Review | レビュー待ち | 自己レビュー完了時 |
| Done | 完了 | 全工程完了時 |

## コミット・完了時の操作

### `mode.source: git` の場合

- ブランチ作成・コミットは通常通り行う
- リモートが設定されていれば `git push` する。なければプッシュをスキップする
- PR 作成は行わない（GitHub 以外のホスティングサービスの場合、手動で PR/MR を作成するよう案内する）

### `mode.source: none` の場合

- Git 操作は一切行わない
- 全工程完了後、変更内容のサマリーを報告して終了する
